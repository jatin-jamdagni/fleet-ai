# Fleet AI — Quick Reference

## One-Command Setup & Deploy

```bash
bash setup.sh
```

That's it. The script asks you questions and handles everything.

---

## What the script configures

| Question | Options |
|----------|---------|
| Deploy mode | `local-docker` · `local-native` · `production` |
| Data mode | `demo` (fake data) · `live` (empty DB) |
| Host / IP | Auto-detected from your network interface |
| Database | Docker-managed · or your own PostgreSQL URL |
| Demo size | How many vehicles and drivers to seed |
| Integrations | Sentry · Resend · Stripe · OpenAI (all optional) |
| Monitoring | Prometheus + Grafana (optional) |

---

## Files generated by setup.sh

```
config/
  fleet.config.ts        ← master config (all settings in one place)

apps/
  backend/.env           ← backend environment variables
  web/.env               ← web VITE_ variables
  mobile/.env            ← mobile EXPO_PUBLIC_ variables

docker-compose.yml       ← local development stack
docker-compose.prod.yml  ← production stack with Nginx

docker/
  nginx/nginx.conf           ← local reverse proxy
  nginx/nginx.prod.conf      ← production Nginx with SSL
  postgres/init.sql          ← DB initialization
  prometheus/prometheus.yml  ← metrics scrape config
  grafana/...                ← provisioning

apps/backend/
  Dockerfile
  src/db/seed.ts         ← demo data seeder

apps/web/
  Dockerfile
  nginx-spa.conf
```

## Manual commands (if you skip auto-deploy)

### Local Docker
```bash
docker compose up -d
docker compose exec backend bunx prisma migrate deploy
docker compose exec backend bun run seed   # demo mode only
```

### Local Native
```bash
# If using Docker just for DB:
docker compose up -d postgres redis

# Backend
cd apps/backend
bun install
bunx prisma migrate deploy
bun run seed      # demo mode only
bun dev

# Web (new terminal)
cd apps/web
bun install
bun dev --host 0.0.0.0
```

### Production
```bash
docker compose -f docker-compose.prod.yml build
docker compose -f docker-compose.prod.yml up -d

# SSL certificate (first time only)
docker compose -f docker-compose.prod.yml run --rm certbot \
  certonly --webroot -w /var/www/certbot \
  -d yourdomain.com \
  --email admin@yourdomain.com --agree-tos
docker compose -f docker-compose.prod.yml restart nginx
```

## Reconfigure

Run `bash setup.sh` again anytime — it overwrites all config files and can redeploy.

To switch from demo → live mode, run again and choose `live`. 
To change your IP (e.g. moved to a different network), run again — it auto-detects.
